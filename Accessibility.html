Accessibility

  • W3C Web Content Accessibility Guidelines are principle- not technology-based
  • 4 Principles:
    • Perceivable
    • Operable
    • Understandable
    • Robust
  • Semantics imporant
  • Browser supporting of accessibility features on HTML5 Accessibility
  • Designing adhering to standards, no flashy cool effects

Putting Code out on the Web

  • Browser sends request to the server (protocol in the address bar is "http" not "file")
  • domain name required
    • bought
  • Hosting service required
    • Gets registered IP address to the domain name
    • vary (free/ mid-range/ full-service)
    • free services: no database; no technical support; advertising
    • e.g. Byet.host

    • File Managing System

    • e.g. "cPanel"
    • Changes are public immediately
    • Versions differ (free/ paid site?)
    • Offers adminitstrative tools (email accounts, database management, cron jobs)
    • Alternative: sftp (to transfer files)
